# 소프트웨어 아키택처

* 모듈화란?
-> 소프트웨어의 성능 향상, 시스템의 수정 및 재사용,
유지 관리 등이 용이하도록 시스템의 기능들을 모듈
단위로 나눈 것을 의미.

* 추상화란?
-> 문제의 전체적이고 포괄적인 개념을 설계한 후 
차례로 세분화하여 구체화시켜 나가는 것.

* 단계적 분해란?
-> 문제를 상위의 중요 개념으로부터 하위의 개념으로
구체화시키는 분할 기법

* 정보 은닉이란?
-> 한 모듈 내부에 포함된 절차와 자료들의 정보가 감추어져
다른 모듈이 접근하거나 변경하지 못하도록 하는 기법

* 소프트웨어의 설계단계 중 상위설계
-> 아키텍처 설계, 예비설계, 시스템의 전체적인 구조

* 소프트웨어의 설계단계 중 하위설계
-> 모듈 설계, 상세 설계, 컴포넌트, 자료구조, 알고리즘

* 소프트웨어 아키텍처 설계 중 시스템 품질 속성
-> 성능, 보안, 가용성, 기능성, 사용성, 변경 용이성, 확장성

* 소프트웨어 아키텍처 설계 중 비즈니스 품질 속성
-> 시장 적시성, 비용과 혜택, 예상 시스템 수명

* 소프트웨어 아키텍처 설계 중 아키택처 품질 속성
-> 개념적 무결성, 정확성, 완결성, 구축 가능성, 변경성, 시험성

* 소프트웨어 아키택쳐의 설계 과정
-> 설계 목표 설정, 시스템 타입 결정, 아키택처 패턴 적용,
서브시스템 구체화, 검토

* 컴포넌트를 설계할 때 클래스에 대한 여러 가정을 공유
할 수 있도록 명세한 것
* 컴포넌트의 오퍼레이션 사용 전 참이 되어야 할 
선행조건, 사용 후 만족되어야 할 결과조건,
오퍼레이션이 실행되는 동안 항상 만족되어야 할 
불변조건이 있음.
-> 협약에 따른 설계
