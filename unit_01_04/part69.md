# 공통 모듈

* 여러 프로그램에서 공통으로 사용할 수 있는 모듈
* 자주 사용되는 계산식이나 매번 필요한 사용자 인증과 같은
기능들이 구성됨
-> 공통 모듈

* 공통 모듈의 재사용 규모에 따른 분류
-> 함수와 객체, 컴포넌트, 애플리케이션

* 공통 모듈 명세 작성 시 시스템 구현 시 해당 기능이 필요하다는 
것을 알 수 있도록 정확히 작성한다는 명세 기법 원칙
-> 정확성

* 공통 모듈 명세 작성 시 일관되게 이해되고 한가지로 해석되도록
작성한다는 명세 기법 원칙
-> 명확성

* 공통 모듈 명세 작성 시 시스템 구현을 위해 필요한 모든
것을 기술한다는 명세 기법 원칙
-> 완전성

* 공통 모듈 명세 작성 시 공통 기능들 간 상호 충돌이 발생하지
않도록 작성한다는 명세 기법 원칙
-> 일관성

* 공통 모듈 명세 작성 시 기능에 대한 요구사항의 출처, 관련
시스템 등의 관계를 파악할 수 있도록 작성한다는 명세 기법 원칙
-> 추적성

* 이미 개발된 기능들을 새로운 시스템이나 기능 개발에 사용하기
적합하도록 최적화하는 작업
-> 재사용

* 명백한 역할을 가지고 독립적으로 존재하는 시스템의 부분
* 인터페이스를 통해 접근할 수 있다
-> 컴포넌트
