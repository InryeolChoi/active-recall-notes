# 객체지향 분석 및 설계

* 객체지향 분석 방법론 중 분석 활동을 객체 모델, 
동적 모델, 기능 모델로 나누어서 설명함
* 모든 소프트웨어 구성요소를 그래픽 표기법을 이용하여
모델링하는 기법
-> 럼바우 방법

* 객체지향 분석 방법론 중 미시적 개발 프로세스와 
거시적 개발 프로세스를 모두 사용함
-> 부치 방법

* 객체지향 분석 방법론 중 유스케이스를 강조하여 사용함
-> jacobson 방법

* 객체지향 분석 방법론 중 E-R 다이어그램을 사용해 객체의 행위를 모델링함
-> Coad와 yourdon 방법

* 객체지향 분석 방법론 중 분석과 설계 간의 구분이 없고, 고객 명세서를 평가해서
설계 작업까지 연속적으로 수행함.
-> Wirfs-Brock 방법

* 럼바우 방법의 분석 활동 순서
-> 객체 모델링, 동적 모델링, 기능 모델링

* 럼바우의 분석기법 중 정보 모델링이라고도 하며, 시스템에서 요구되는 객체를 찾아내어
속성과 연산 식별 및 객체를 찾아내어 속성과 연산 식별 및 객체들
간의 관계를 규정하여 객체 다이어그램으로 표시하는 것
-> 객체 모델링

* 럼바우의 분석기법 중 상태 다이어그램을 이용하여 시간의 흐름에 따른 객체들
간의 제어 흐름, 상호 작용, 동작 순서 등의 동적인 행위를 표현하는 
모델링
-> 동적 모델링

* 럼바우의 분석기법 중 자료 흐름도를 이용하여 다수의 프로세스들 간의
자료 흐름을 중심으로 처리 과정을 표현한 모델링
-> 기능 모델링

* 객체지향 설계원칙 5가지
-> 단일 책임 원칙, 개방-폐쇄 원칙, 리스코프 치환 원칙, 
인터페이스 분리 원칙, 의존 역전 원칙
