# UML - 관계

* 사물과 사물 사이의 연관성을 표현하는 것
-> 관계

* 2개 이상의 사물이 서로 관련되어 있는 관계
-> 연관관계
-> Association

* 하나의 사물이 다른 사물에 포함되어 있는 관계
-> 집합관계
-> Aggregation

* 포함하는 사물의 변화가 포함되는 사물에 영향을 미치는 관계
-> 포함관계
-> Composition

* 하나의 사물이 다른 사물에 비해 더 일반적이거나 
구체적인 관계
-> 일반화 관계
-> Generalization

* 사물 사이에 서로 연관은 있으나 필요에 의해 서로에게 
영향을 주는 짧은 시간 동안만 연관을 유지하는 관계
-> 의존 관계
-> Dependency

* 사물이 할 수 있거나 해야 하는 기능
-> 실체화 관계
-> realization
