# S/W 공학의 발전적 추세

* 전자 칩과 같은 소프트웨어 부품, 즉 블록을 만들어서
끼워 맞춰 소프트웨어를 완성시키는 방법
-> 합성 중심

* 추상화 명세로 써진 명세를 구체화하여 프로그램을 만드는 방법,
패턴 구성 방법이라고도 함
-> 생성 중심

* 소프트웨어 재공학의 개념
-> 새로운 요구에 맞도록 기존 시스템을 이용하여 보다 나은
시스템을 구축하고, 새로운 기능을 추가하여 소프트웨어 성능을
향상시키는 것

* 소프트웨어 개발 과정에서 사용되는 요구분석, 설계, 구현, 검사 및
디버깅 과정 전체 또는 일부를 컴퓨터와 전용 소프트웨어 도구를 사용하여
자동화하는 것
-> CASE

* 이미 개발되어 인정받은 소프트웨어의 전체 혹은 일부분을 다른
소프트웨어 개발이나 유지에 사용하는 것
-> 소프트웨어 재사용
