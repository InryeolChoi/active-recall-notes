# UML - 다이어그램

* 사물과 관계를 도형으로 표현한 것
-> 다이어그램

* 클래스에 속한 사물들, 즉 인스턴스를
 특정 시점의 객체 간 과녜로 표현
* 럼바우 객체지향 분석 기법에서
객체 모델링에 활용
-> 객체 다이어그램

* 실제 구현 모델인 컴포넌트 간 관계나
컴포넌트 간 인터페이스를 표현함
-> 컴포넌트 다이어그램

* 결과물, 프로세스, 컴포넌트 등 
물리적 요소들의 위치를 표현함
-> 배치 다이어그램

* 클래스나 컴포넌트가 복합 구조를 갖는 경우
그 내부구조를 표현함
-> 복합체 구조 다이어그램

* 유스케이스나 클래스 등의 모델 요소를 그룹화한
패키지들의 관계를 표현함
-> 패키지 다이어그램

* 사용자의 요구를 분석하는 것으로, 기능 모델링 작업에 사용함
-> 유스케이스 다이어그램 

* 상호 작용하는 시스템이나 객체들이 주고받는 메시지를 표헌함
-> 순차 다이어그램

* 동작에 참여하는 객체들이 주고받는 메세지와 객체들 간의
연관 관계를 표현함
-> 커뮤니케이션 다이어그램

* 하나의 객체가 자신이 속한 클래스의 상태 변화 혹은 다른
객체와의 상호 작용에 따라 상태가 어떻게 변하는지를 표현함.
* 럼바우, 동적 모델링
-> 상태 다이어그램
