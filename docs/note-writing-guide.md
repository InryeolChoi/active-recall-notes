# 노트 작성 가이드

`active-recall-notes`의 markdown은 단순한 메모가 아니라, 이후 `active-recall-quiz`에서 문제와 복습 데이터로 활용될 수 있는 원문이다.
따라서 처음부터 "읽기 좋은 문서"이면서 동시에 "문제로 꺼내기 좋은 문서"를 목표로 작성한다.

## 기본 방향

- 개념명은 분명하게 드러나도록 제목 또는 소제목으로 분리한다.
- 정의, 특징, 목적, 장단점, 비교 포인트처럼 문제로 만들기 쉬운 단위로 끊어서 정리한다.
- 문장은 짧게 쓰되, 답안으로 옮겨 적어도 어색하지 않도록 정확한 표현을 사용한다.
- 지나치게 압축된 키워드 나열보다, 회상 가능한 완전한 문장을 우선한다.

## 권장 구성

하나의 주제를 정리할 때는 아래 흐름을 기본 템플릿으로 삼는다.

## 개념명

### 정의

개념이 무엇인지 한두 문장으로 설명한다.

### 왜 중요한가

시험에서 왜 자주 다뤄지는지, 또는 어떤 맥락에서 필요한지 적는다.

### 핵심 포인트

- 반드시 기억해야 하는 특징
- 헷갈리기 쉬운 포인트
- 다른 개념과 비교되는 부분

### 서술형 답안 힌트

서술형으로 답할 때 포함하면 좋은 키워드나 문장 흐름을 적는다.

## 작성 시 주의할 점

- 너무 긴 문단은 피하고, 한 문단에는 하나의 핵심만 담는다.
- 비슷한 개념은 비교 기준을 함께 적어 혼동을 줄인다.
- 암기용 약어만 적지 말고, 풀어서 설명하는 문장도 함께 남긴다.
- 추후 자동 저장과 활용을 고려해 markdown 구조를 최대한 단순하고 일관되게 유지한다.

## 지향하는 결과

좋은 노트는 "읽으면 이해되는 문서"를 넘어서, "문제를 봤을 때 답이 떠오르는 문서"여야 한다.
이 저장소의 모든 markdown은 그 목적에 맞춰 누적된다.
