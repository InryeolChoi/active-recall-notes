# 인터페이스 요구사항 검증

* 요구사항 명세서의 오류 확인 및 표준 준수 여부
등의 결함 여부를 검토 담당자들이 수작업으로 분석하는 방법
-> 요구사항 검토

* 요구사항 명세서 작성자가 명세서 내용을 직접 설명하고
동료들이 이를 들으며 결함을 발견하는 형태의 검토방법
-> 동료검토

* 검토 회의 전 요구사항 명세서를 미리 배포하여 사전
검토한 후에 짧은 검토 회의를 통해 결함을 발견하는
형태의 검토 방법
-> 워크쓰루

* 요구사항 명세서 작성자를 제외한 다른 검토 전문가들이
요구사항 명세서를 확인하면서 결함을 발견하는 형태의 
검토 방법
-> 인스펙션

## 인터페이스 요구사항 검증의 주요 항목
* 요구사항이 어떻게 보다 무엇을 에 중점을 두고 있는가
-> 기능성

* 요구사항이 사용자의 요구를 모두 만족하고, 개발된 
소프트웨어가 사용자의 요구 내용과 일치하는지를 검증할
수 있는가
-> 검증 가능성
